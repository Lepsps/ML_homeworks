# Домашняя работа №3: Кластеризация данных банковского маркетинга

## Описание проекта

Данный проект выполнен в рамках домашнего задания по курсу "Машинное обучение". Целью работы является применение методов обучения без учителя (кластеризации) для поиска скрытых структур в данных банковских клиентов. Работа реализована в виде Jupyter ноутбука и сравнивает полученные кластеры с реальными исходами маркетинговой кампании.

## Задача

**Бизнес-контекст:**
Банк хочет понять, существуют ли естественные группы клиентов с похожим поведением и характеристиками. Если алгоритмы кластеризации смогут автоматически выделить группу клиентов, склонных к открытию депозита, это позволит автоматизировать сегментацию без необходимости разметки данных вручную.

**Задача машинного обучения:**
1. Выполнить кластеризацию клиентов, **не используя** целевую метку `deposit` (обучение без учителя).
2. Визуализировать данные в пространстве пониженной размерности.
3. Сравнить полученные кластеры с истинными метками, чтобы понять, совпадают ли найденные алгоритмом группы с реальным поведением клиентов.

## Используемый датасет

Для решения задачи был использован датасет **"Bank Marketing Dataset"**. Он содержит информацию о клиентах португальского банка, их демографические данные и историю взаимодействия.

**Ссылка на датасет:** [https://www.kaggle.com/datasets/janiobachmann/bank-marketing-dataset/data](https://www.kaggle.com/datasets/janiobachmann/bank-marketing-dataset/data)

Данные загружаются в ноутбук напрямую по URL.

## Этапы работы

Ноутбук содержит следующие шаги:

1.  **Загрузка и подготовка данных:** 
    *   Чтение данных.
    *   **Важно:** Отделение целевой переменной `deposit`. Она сохраняется *только* для финальной валидации и не участвует в обучении моделей.
2.  **Предобработка:**
    *   Кодирование категориальных признаков (`OneHotEncoder`).
    *   Масштабирование данных (`StandardScaler`), так как алгоритмы кластеризации чувствительны к дистанциям между точками.
3.  **Понижение размерности (PCA):**
    *   Использование метода главных компонент (PCA) для сжатия пространства признаков до 2-х измерений.
    *   Визуализация истинного распределения классов на плоскости.
4.  **Кластеризация (3 метода):**
    Применены следующие алгоритмы:
    *   **K-Means (k-средних):** С принудительным разбиением на 2 кластера.
    *   **DBSCAN:** Плотностный алгоритм кластеризации (автоматическое определение числа кластеров и поиск шума).
    *   **Agglomerative Clustering:** Иерархическая кластеризация.
5.  **Оценка и Визуализация:**
    *   Сравнение полученных меток кластеров с истинными метками (`yes`/`no`) с использованием метрик **ARI** (Adjusted Rand Index) и **V-Measure**.
    *   Визуализация результатов работы каждого алгоритма в осях PCA.

## Результаты

В ходе эксперимента было выявлено, что признаки клиентов не образуют четко разделяемых геометрических кластеров, которые соответствовали бы целевому действию (открытию депозита). 

*   **Визуализация PCA:** Показывает сильное перекрытие классов `yes` и `no`.
*   **Метрики:** Значения ARI для всех алгоритмов близки к 0 (K-Means: ~0.03), что говорит об отсутствии корреляции между естественными кластерами данных и целевой переменной.
*   **Вывод:** Для задачи прогнозирования депозита методы обучения без учителя (кластеризация) оказались неэффективны. "Естественная" группировка клиентов происходит по иным признакам, не связанным напрямую с согласием на депозит. Для этой задачи следует использовать методы обучения с учителем (Classification).
